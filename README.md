# german-tutor

A multi-agent **CEFR German tutor (A1–B2)** for the terminal, built with
**[LangGraph](https://langchain-ai.github.io/langgraph/)**. It teaches paced
lessons, generates and grades exercises, drills vocabulary with spaced repetition,
runs conversation practice, places your level, plays **pronunciation audio**, and
**remembers where you left off** between sessions.

---

## Highlights

- **Multi-agent state graph.** A router classifies each turn and dispatches to one
  specialist ReAct agent (placement, lesson, grammar, vocab, exercise, conversation,
  progress, concierge). The router doubles as an on-topic guardrail.
- **Two-layer persistence.**
  - *Conversation / graph state* → LangGraph's **SQLite checkpointer**, keyed by
    `thread_id = learner id`. Re-running resumes the exact graph state, including an
    in-flight lesson.
  - *Durable learning progress* → a separate SQLite `Store`: level, per-unit mastery,
    a **lesson pointer** (resume mid-lesson), the **spaced-repetition deck**, attempt
    history, and an error log that drives targeted review.
- **Fixed curriculum + generated, cached delivery.** Units, objectives, grammar
  points and vocab themes are static data (`curriculum/*.yaml`). Lesson *delivery* is
  generated by the model but **cached per unit**, so resuming replays the identical
  lesson while extra practice stays fresh.
- **Spaced repetition (SM-2).** Vocabulary is scheduled with the classic SuperMemo-2
  algorithm (`srs.py`).
- **Pronunciation audio.** Uses the macOS `say` command (native German voice) — no
  API key. German words render as `🔊N` markers; play them with `/play N`.
- **Adaptive & honest.** Difficulty follows performance; recurring mistakes become
  review fuel; the tutor stays at your CEFR level and won't invent grammar rules.

## Architecture

```
START ─► router ──(conditional)──► placement ─┐
                                   lesson ─────┤
                                   grammar ────┤
                                   vocab ──────┼─► END
                                   exercise ───┤
                                   conversation┤
                                   progress ───┤
                                   concierge ──┤
                                   offtopic ───┘   (refusal / guardrail)
```

Each specialist is a `create_react_agent` with only the tools it needs. Tools
(`tools_lc.py`) read/write the durable `Store`; the checkpointer persists graph state.

## Requirements

- **Python ≥ 3.11**, [`uv`](https://docs.astral.sh/uv/)
- An **OpenAI API key**
- macOS for pronunciation audio (`say` / `afplay`) — optional; the tutor works
  without it.

## Setup

```bash
uv sync
cp .env.example .env      # set OPENAI_API_KEY=sk-...
```

| Variable | Default | Purpose |
|----------|---------|---------|
| `OPENAI_API_KEY` | — | required |
| `OPENAI_MODEL` | `gpt-5.4` | chat model |
| `TUTOR_LEARNER_ID` | `default` | stable id → progress + resume |
| `TUTOR_DB` | `.german_tutor/progress.db` | progress database path |
| `TUTOR_TTS_VOICE` | `Anna` | German voice for audio |
| `TUTOR_TTS_AUTOPLAY` | `1` | set `0` to generate audio without auto-playing |
| `TUTOR_WEB_HOST` / `TUTOR_WEB_PORT` | `127.0.0.1` / `8000` | web GUI bind address |

## Run

```bash
uv run german-tutor                       # learner id 'default'
uv run german-tutor --learner-id anna     # separate profile + thread
```

In-chat commands:

| Command | Action |
|---------|--------|
| `/lesson` | Start or continue a lesson |
| `/resume` | Resume your previous lesson |
| `/review` | Review vocab due today (SRS) |
| `/quiz` | Practice exercises for your unit |
| `/practice` | Have a German conversation |
| `/placement` | Placement check to set your level |
| `/progress` | See how you're doing |
| `/play N` | Play the Nth 🔊 word from the last reply aloud |
| `/level A1\|A2\|B1\|B2` | Set your level directly |
| `/whoami`, `/help`, `/quit` | — |

### Resume in action

```
$ uv run german-tutor --learner-id anna

You: /resume

```

## Web GUI

A browser GUI is included where each German word is a **clickable 🔊 play button**
(real click-to-play, unlike the terminal):

```bash
uv run german-tutor-web        # then open http://127.0.0.1:8000
```

It wraps the same LangGraph tutor and shares the same persistence (resume works
across the CLI and GUI). Needs `OPENAI_API_KEY` for chat and macOS `say` for audio.
Override the bind address with `TUTOR_WEB_HOST` / `TUTOR_WEB_PORT`.

Endpoints: `POST /api/chat`, `GET /api/state`, `GET /api/tts?text=…&voice=Anna`.

## Testing

Fast, LLM-free unit tests cover the deterministic core (SRS scheduling, curriculum
lookup, progress persistence/resume, TTS synthesis):

```bash
uv run pytest
```

Agent **behavior** is validated and improved with **[RELAI](https://relai.ai)** —
simulated learners drive the tutor and LLM judges score the result.

### Learning environments

Three lesson-quality environments live in `.relai/learning-envs/` (tag `end-to-end`):

- `beginner-lesson-detail` — detailed, well-paced first lesson
- `detailed-no-german-assumed` — assumes zero prior German; everything explained
- `high-quality-lesson` — clear objective, examples, level-appropriate, checks understanding

```bash
relai simulate --tags end-to-end --env-file .relai/simulator.env                       # score current behavior
relai optimize --tags end-to-end --env-file .relai/simulator.env --total-rollouts 30   # improve it
```

### How the optimizer improved the agent

A `relai optimize` run (30 rollouts) accepted one change — *"Improve absolute-beginner
lesson openings"*:

- **Lesson prompt:** for a first lesson, teach for **zero prior knowledge** — gloss
  every German word inline in English, no unexplained "preview" German, explicit
  pronunciation hints and tasks.
- **`get_learner_state`** now returns an **`is_absolute_beginner`** flag so the lesson
  agent reliably detects a true first lesson and switches into detailed mode.

### Mocked data for testing

RELAI decides per tool whether to **mock** (deterministic stub) or run **live**
(`.relai/mock-manifest.json`):

| Policy | Tools |
|--------|-------|
| **mock** (state-mutating writes) | `set_level`, `save_lesson_pointer`, `cache_lesson`, `record_attempt`, `log_error`, `update_mastery`, `add_vocab`, `review_vocab`, `log_session_summary` |
| **do_not_mock** (safe reads, live) | `get_learner_state`, `get_next_unit`, `get_unit_details`, `get_cached_lesson`, `get_due_vocab` |

Mutating tools are mocked so simulations don't corrupt real learner state; read-only
tools run live so the agent sees the real curriculum/progress. Learning environments
provide deterministic outputs for mocked tools via their `mocks={}` field. The LLM
model calls run live (not mocked).

## Notes & limitations

- Two front-ends: the **CLI** (text + `/play` audio) and the **web GUI** (clickable
  🔊 buttons) — see *Web GUI* above.
- Lessons are generated then cached; `/review` and extra practice are intentionally
  fresh each time, while a unit's core lesson stays stable across resumes.
- Local RELAI files (`.relai/config.toml`, `.relai/simulator.env`, runs, caches) and
  `.env` are gitignored; collaborators add their own `.env` and run `relai init`.
